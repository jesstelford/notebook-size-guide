# CLAUDE.md

Guidance for an agentic coding tool continuing work on this project.

## Project

An interactive, **to-scale notebook-size reference guide**: a single static page that
compares notebook/paper dimensions across ISO, JIS, and brand/format sizes, with
region-aware shopping links.

- **Live:** https://jesstelford.github.io/notebook-size-guide/
- **Repo:** https://github.com/jesstelford/notebook-size-guide
- **All code is in `docs/index.html`** — one self-contained file (HTML + CSS + vanilla JS).
  No framework, no bundler, no build step.

## Running & deploying

- **Local:** open `docs/index.html` in a browser, or serve statically (`npx serve docs`). No install/build.
- **Deploy:** GitHub Pages, `main` / `docs` folder (Settings → Pages → source "Deploy from a
  branch", folder `/docs`). `docs/index.html` is the entry point.
- **Runtime deps (CDN, not vendored):** Google Fonts (Space Grotesk, IBM Plex Sans/Mono)
  and one client-side `fetch` to `https://ipwho.is/` for region detection. No other network
  calls, no server, no secrets.

## Hard constraints (keep these)

- **Single file.** Keep everything in `docs/index.html` unless there's a strong reason to split.
- **No browser storage.** Do not use `localStorage` / `sessionStorage` / cookies — state is in-memory only.
- **No build tooling** unless explicitly requested; it must stay openable as a bare file.
- Dependency-light and framework-free.

## Code style

- Plain browser JS in the inline `<script>` — no transpile step, no modules, no framework.
- Grouped by concern (see the script order above). Prefer deriving state from a single
  source of truth over tracking it separately — e.g. the filter buttons' selected state is
  derived from the `hidden` set, not stored.

## Layout inside `docs/index.html`

1. `<head>` — meta, fonts, **all CSS**. CSS variables define an "ink-on-paper / drafting
   spec-sheet" theme.
2. `<body>`:
   - `header` (eyebrow + `h1`)
   - `.stage > svg#diagram` — the to-scale drawing
   - `.stage-foot` — `#filters` button row + `.foot-right` (`.region` selector + `.baseline` toggles)
   - `.cards#cards` — per-size info cards (generated by JS)
   - `.notes` — footnote cards (JIS vs ISO, Slim, shopping links, brand-size caveats)
3. `<script>` (all logic), in this order:
   - **Marketplace/region:** `MARKETS`, `MARKET`, `amazonURL` / `camelURL`
   - **Units:** `IMPERIAL`, `UNIT`, `unitForMarket`, `fmtDim`, `updateUnits`
   - **Detection:** `countryFromLocale`, `detectCountry`
   - **Data:** `sizes[]`
   - **Diagram:** `renderDiagram()`
   - **Cards:** `prodHTML`, `cardHTML`, `cardOrder`, render + checkbox wiring
   - **Filters:** `GROUPS`, `syncChecks`, `refreshFilters`, click handler
   - **Region wiring:** `regionSel`, `baselines`/`syncBaselineButtons`, `applyMarket`, detection init

## Data model

`sizes` is the **single source of truth** (array of objects). Fields:

- `id` — unique string; used by checkboxes, filters, and the diagram.
- `name`, `tag` — display name + pill text (e.g. `"B6"` / `"JIS"`,
  `"Traveler's"` / `"Regular · non-standard"`). The part of `tag` before the first `·`
  is the short label used in diagram callouts.
- `color` — hex; the size's ink-wash / leader-line / pill colour.
- `w`, `h` — **real width/height in millimetres**. Drive the to-scale drawing and all unit conversion.
- `mm` — human string incl. unit; shown as the secondary reference (may contain `≈`).
- `in` — legacy inch string, **no longer displayed** (`fmtDim` computes units live).
- `alt` — small grey descriptor line.
- `use` — one-line use case.
- `approx` (optional bool) — prepends `≈` to computed dimension strings.
- `none` (optional HTML) — honest note shown when the size has no clean match (Slim / Reporter's).
- `products[]` — `{ brand, model, size, specs: [[k,v], …], note?, search }`.
  `search` is a **market-neutral** query string used to build both Amazon and Camel links.

**Current 20 sizes** (cards render tallest-first to match the diagram callout order):
A4, B5 JIS, B5 ISO, B5 Slim (approx), Traveler's Regular, A5, Moleskine Large,
Reporter's (approx), Baron Fig Confidant (approx), Hobonichi Weeks, B6 JIS,
B6 Slim Stalogy, B6 ISO, B6 Slim Midori, A6, Field Notes, B7 JIS, Traveler's Passport,
B7 ISO, A7.

- **Moleskine Large** (130×210) — A5-height but 18 mm narrower; covers the whole Moleskine
  Large line (Classic, Cahier) and the fountain-pen "A5 Slim" 130×210 footprint.
- **Baron Fig Confidant** (≈137×196, from a 5.4×7.7 in spec) — smaller than A5 on both axes.
- **Hobonichi Weeks** (94×188) — the only bespoke Hobonichi trim (not A/B); Weeks Mega shares
  it. The A6 Original and A5 Cousin reuse existing A6/A5 entries (listed there as products).
- **A7** (74×105 ISO) and **B7** (JIS 91×128 / ISO 88×125). Genuine ISO-B7 notebooks barely
  exist at retail, so that entry links out to the closest brand substitute (Rhodia No. 12).

`MARKETS` maps an ISO country code → `{ code, label, flag, amazon (host), camel (subdomain
prefix | null) }`. `camel: null` ⇒ Camel unsupported in that marketplace ⇒ the Camel link is
hidden. `XX` is the ".com / other" fallback.

## Core behaviours

- **Diagram (`renderDiagram`)** — all visible sizes are translucent rectangles sharing a
  bottom-left corner (largest drawn behind). The `viewBox` is **recomputed each render from
  the visible bounding box**, so hiding larger sizes zooms the rest to fill the container.
  A right-hand gutter holds non-overlapping leader-line callouts (name + dimensions), ordered
  top-to-bottom by height with a push-down anti-overlap pass. To-scale **baseline references**
  are drawn at the shared bottom-left corner: a filled dark **credit card (85.6 × 54 mm)** and/or
  an outlined **US Letter sheet (215.9 × 279.4 mm / 8.5 × 11 in)**. The active set lives in the
  `baselines` Set (`"card"` / `"letter"`), toggled by the Baseline buttons; either, both, or
  neither can show. Active refs are factored into the bounding-box computation so they never clip
  when only small sizes are visible. Callout dimensions use the active unit.
- **Cards** — generated from `cardOrder` (= `sizes` sorted by height, descending). Each card:
  visibility checkbox (updates the `hidden` Set + diagram), inline header (checkbox + name +
  tag pill), dimensions (switchable primary unit + mm secondary), use case, and a collapsible
  product section.
- **Filters (`GROUPS` + `refreshFilters`)** — additive multi-select with two modes:
  - **All mode** (`hidden.size === 0`): the `All` button is active, group buttons off.
  - **Partial mode**: `All` off; a group button is active **iff every item in it is visible**.
  - Button state is **derived** from `hidden` on every change — never tracked separately.
  - `All` only ever turns **ON**. The first group click from All mode focuses down to that
    group; subsequent clicks are additive (toggle whole categories on/off). Individual
    checkbox changes re-derive all button states.
  - Groups: `a4`, `a5` (A5 + Moleskine Large + Baron Fig), `b5` (3 variants),
    `b6` (4 variants), `b7` (JIS + ISO), `a6`, `a7`,
    `ftr` (Field Notes + Traveler's Regular/Passport + Reporter's + Hobonichi Weeks).
- **Baseline (`baselines` + `syncBaselineButtons`)** — a pair of independent toggle buttons
  (Credit Card / US Letter) next to the Region selector, controlling which to-scale references
  the diagram draws. `applyMarket()` sets the region default (US → Letter, else Card); the
  buttons then override, and both may be on at once.
- **Region + units** — `detectCountry()` tries `ipwho.is` (non-intrusive; **no** geolocation
  permission prompt), falls back to browser locale, then `XX`. `applyMarket()` sets the
  marketplace (Amazon + Camel link domains) and the unit (**US/CA → inches, everywhere else →
  cm**), then refreshes links, dimension text, the baseline default (US → Letter, else Card),
  and the diagram. A dropdown allows manual override; a manual choice suppresses later
  auto-detection.

## Design principles & decisions (the "why")

- **Accuracy over convenience — never fabricate data.** No invented prices, ratings, review
  counts, or specs. Amazon and Camel block automated reads and search snippets don't expose
  live prices, so pricing is **linked out, not embedded**. (An embedded price-snapshot feature
  was built and then deliberately **removed** for this reason.) If a value can't be verified,
  show an honest "not available" / link out — do not guess.
- **JIS is the primary B-series standard** (these are Japanese-notebook sizes); ISO is a
  separate entry/annotation. A-sizes are identical worldwide (one entry each).
- **"Slim" is not a real standard** — brand-specific (Midori ≈110×176, Stalogy 100×180);
  B5 Slim barely exists (Semi-B5 substitute noted). Hardcover-dotted doesn't exist in Slim,
  so softcover/grid options are accepted there.
- **"Reporter's" is a binding style, not a fixed size** — flagged `approx`; products list
  their own real dimensions.
- **Brand sizes ≠ pure ISO** — Leuchtturm's Pocket / B6+ / Composition / Master run off-spec;
  always display each product's **real** dimensions rather than implying a match.
- **Price aggregator = CamelCamelCamel.** Camel covers US/UK/CA/DE/FR/IT/ES/JP/**AU**;
  **Keepa does not cover Australia**, which is why Camel was chosen. Camel tracks price only
  (not ratings), so ratings/reviews are left to the Amazon listing.

## Known limitations / gotchas

- Region detection depends on `ipwho.is`; if it's blocked or rate-limited it silently falls
  back to locale — expected, not a bug.
- Country-flag emoji in the region `<select>` render as letters on some platforms (Windows) —
  cosmetic only.
- Product prices/ratings are intentionally **not** in the page — links only.
- The Camel link is hidden for marketplaces Camel doesn't support.
